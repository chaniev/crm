# TASK-020: Довести empty states и причины недоступности действий в карточке клиента

## Status
risky

## Goal
Пустые состояния ведут пользователя к следующему действию, а disabled actions объясняют конкретную причину недоступности.

## Context
Нужно сделать action-first empty states: нет абонемента -> оформить, нет посещений -> отметить первое, клиент не в группе -> добавить, контакты не добавлены -> добавить контакт, фото не загружено -> компактная кнопка загрузки. Для отключенных кнопок нужны причины: нет прав, нет текущего абонемента, клиент в архиве, действие недоступно для текущего статуса. Общий текст про management-роли заменить подсказками около конкретных действий.

## User role
администратор / тренер

## Problem
Пользователь видит пустые или недоступные состояния, но не понимает, что можно сделать дальше и почему действие заблокировано.

## Scope
- Найти empty states в client detail.
- Заменить их на компактные action-first states.
- Добавить объяснения недоступности около конкретных actions.
- Убрать общий текст про management-роли там, где нужна конкретная причина.

## Out of scope
- Изменение backend permission rules.
- Изменение статусов клиента, абонемента или архива.
- Добавление новых действий, не перечисленных в source note.

## Constraints
- Backend владеет permissions и validation semantics.
- Frontend может показывать только причины, основанные на backend state/contract или уже существующих UI constraints.
- Нельзя скрывать действие так, чтобы пользователь не понимал, почему оно недоступно.

## Acceptance criteria
- [ ] Empty states предлагают конкретное действие для абонемента, посещений, группы, контактов и фото.
- [ ] Disabled buttons показывают понятную причину рядом с действием.
- [ ] Общий текст про management-роли заменен контекстными подсказками.
- [ ] UI не ломается на mobile.

## Test checklist
- [ ] Запустить `cd frontend && npm run lint`.
- [ ] Запустить `cd frontend && npm run build`.
- [ ] Проверить пользователя без прав и пользователя с правами.
- [ ] Проверить клиента без абонемента, без посещений, без группы, без контактов и без фото.
