# TASK-002: Довести отображение абонементов в Telegram-боте

## Status
risky

## Goal
Пользователь Telegram-бота видит достаточные данные по абонементам и оплате, чтобы не принимать решение вслепую.

## Context
В списке заканчивающихся абонементов нужно показывать количество дней до окончания. В подтверждении оплаты нужно показывать ФИО, тип абонемента, дату покупки и дату окончания.

## User role
администратор / тренер

## Problem
Текущие Telegram-сценарии по абонементам и оплате показывают недостаточно контекста и могут привести к ошибочным действиям.

## Scope
- Обновить сообщение списка заканчивающихся абонементов.
- Добавить расчет или получение количества дней до окончания без расхождения с backend-правилами.
- Обновить текст подтверждения оплаты.
- Покрыть сценарии тестами на формат и обязательные поля.

## Out of scope
- Изменение правил действия абонементов.
- Изменение финансовой логики оплаты.
- Добавление новых типов абонементов.

## Constraints
- Backend остается источником истины по memberships, subscriptions и payment semantics.
- Бот не должен вычислять доменные правила, если backend уже отдает нужные данные.
- Даты должны быть однозначными и проверяемыми в тестах.

## Acceptance criteria
- [ ] В списке заканчивающихся абонементов видно количество дней до окончания.
- [ ] Подтверждение оплаты содержит ФИО, тип абонемента, дату покупки и дату окончания.
- [ ] Формат не ломается на отсутствующих или частичных данных.
- [ ] Добавлены или обновлены scenario tests.

## Test checklist
- [ ] Запустить `cd bot && ruff check .`.
- [ ] Запустить `cd bot && pytest`.
- [ ] Вручную проверить Telegram-сценарий списка абонементов и подтверждения оплаты.
