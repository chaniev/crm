# TASK-026: Уточнить статистику и финансовые отчеты

## Status
needs-clarification

## Goal
Сформировать понятный набор статистических и финансовых отчетов с определениями метрик, периодами и фильтрами.

## Context
В inbox есть заметка: `Статистика (количество проданных абонементов по филиалам, количество новых клиентов, финансовый отчет (за мес, год)...)`.

Уточнение от 2026-05-07:
- расчет по тренеру должен строиться на том, сколько абонементов было куплено на тренировки конкретного тренера.

## User role
владелец / администратор

## Problem
Запрос на статистику слишком широкий: неизвестны точные метрики, фильтры, права доступа, формулы и место в интерфейсе.

## Scope
- Уточнить список отчетов и метрик.
- Зафиксировать метрику по тренеру: количество купленных абонементов на тренировки конкретного тренера и производные финансовые показатели.
- Уточнить периоды: месяц, год, произвольный диапазон.
- Уточнить фильтры: филиал, тренер, группа, тип абонемента.
- Уточнить источник связи `покупка абонемента -> тренировки конкретного тренера`.
- Уточнить правила финансового отчета.
- После ответов разбить на отдельные risky implementation tasks.

## Out of scope
- Немедленная реализация отчетов.
- Расчет финансовых метрик без согласованных определений.

## Constraints
- Backend владеет financial report semantics.
- Расчет по тренеру должен выполняться backend-логикой на основе согласованной связи абонемента с тренировками конкретного тренера.
- Филиалы пока не уточнены и могут быть dependency.
- Нельзя дублировать финансовые формулы во frontend.

## Acceptance criteria
- [ ] Есть список отчетов первого релиза.
- [ ] Для каждой метрики есть формула и источник данных.
- [ ] Определены периоды, фильтры и роли доступа.
- [ ] Созданы отдельные implementation tasks.

## Test checklist
- [ ] После будущей реализации проверить backend report tests.
- [ ] После frontend реализации проверить lint/build и UI на пустых данных.

- Risk level: high
- Reason: финансовые отчеты, филиалы и аналитика требуют точных бизнес-определений.
