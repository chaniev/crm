# TASK-022: Уточнить продуктовую модель филиалов

## Status
done

## Goal
Понять, что означает `филиал` в CRM и как он влияет на клиентов, группы, тренеров, расписание, абонементы и отчеты.

## Context
В inbox есть короткая заметка: `Добавить филиалы`.

Уточнение от 2026-05-07:
- Филиалы задаются в настройках CRM.
- Обязательные поля филиала: название.
- Дополнительные поля филиала: адрес, описание.
- Филиал можно архивировать.
- Клиент может быть прикреплен только к одному филиалу.
- У тренера нет прямой привязки к филиалам.
- Тренер видит только клиентов в своих группах.
- Секции и группы также привязаны к филиалам.
- Группа может быть привязана только к одному филиалу.
- Внутри филиалов должен быть список залов.
- Поля зала: название, описание.
- Зал нельзя удалить или архивировать, если к нему привязаны группы или расписание.
- Администратор видит все филиалы и может добавлять/редактировать филиал.
- Нельзя добавить клиента из одного филиала в группу другого филиала.
- У тренера нет отдельной проверки прямой привязки к филиалу: филиалы тренера определяются через назначенные ему группы.
- Тренер не может вести группу, на которую он не назначен.
- При переводе клиента в другой филиал абонемент сохраняется, клиент удаляется из текущих групп и остается без группы, если администратор в процессе переноса не выбрал группу из нового филиала.
- Миграция существующих данных не нужна: деплой будет с нуля.
- Абонемент привязан к филиалу клиента.
- Отчеты по филиалам нужно вынести в отдельную задачу.

## Problem
Ключевые правила по филиалам и залам уточнены. Реализация вынесена в отдельные risky-задачи.

## Scope
- Зафиксировать связи филиалов с клиентами, тренерами, секциями, группами и абонементами.
- Зафиксировать справочник залов внутри филиала.
- Зафиксировать, как филиалы и залы редактируются в настройках CRM.
- Зафиксировать влияние филиалов на видимость данных и правила валидации.
- Зафиксировать, что миграция существующих данных не требуется.
- Разложить на backend/frontend/bot/reporting follow-up tasks.

## Out of scope

## Constraints

- Добавление филиалов может потребовать migrations и contract changes.
- Нужно обновить всех потребителей при изменении backend contract.
