# Задачи: оставшиеся улучшения окна клиентов

Дата разнесения: 2026-05-06

Источник: бывший `docs/CLIENTS_WINDOW_IMPROVEMENT_PLAN.md`.

## Карточка клиента

1. Сделать верх карточки клиента более компактным:
   - ФИО;
   - телефон;
   - статус;
   - текущий абонемент;
   - дата окончания;
   - последнее посещение;
   - группа.

2. Довести быстрые действия в карточке:
   - отметить посещение;
   - новый абонемент;
   - принять оплату;
   - добавить в группу;
   - редактировать;
   - отправить в архив.

3. Сделать возврат к списку клиентов с сохранением поиска, фильтров, выбранного клиента и позиции в списке.

## Уменьшение визуального шума в detail/form сценариях

1. Проверить специализированные карточки client form/detail на лишние пустые блоки.
2. Большой просмотр фото показывать только если фото уже загружено или пользователь явно открыл просмотр.
3. Убрать повторяющиеся статусы и не дублировать одно и то же состояние в нескольких блоках.
4. Сделать данные клиента в карточке более плотными, не теряя читаемость.

## Вкладки карточки клиента

Разделить карточку на вкладки:

1. `Обзор`:
   - ключевые данные;
   - текущий абонемент;
   - последние посещения;
   - быстрые действия.
2. `Абонемент и оплата`:
   - текущий абонемент;
   - история абонементов;
   - оплаты;
   - продление и исправление.
3. `Посещения`:
   - история посещений;
   - фильтр по периоду;
   - отметка посещения.
4. `Группы`:
   - текущие группы;
   - добавление в группу;
   - удаление из группы.
5. `Контакты`:
   - контактные лица;
   - телефон;
   - дополнительные контакты.
6. `История`:
   - изменения карточки;
   - версии абонемента;
   - служебные события.

## Empty states и причины недоступности действий

1. Довести пустые состояния detail-сценариев до action-first поведения:
   - `Нет абонемента` -> `Оформить абонемент`;
   - `Нет посещений` -> `Отметить первое посещение`;
   - `Клиент не в группе` -> `Добавить в группу`;
   - `Контакты не добавлены` -> `Добавить контакт`;
   - `Фото не загружено` -> компактная кнопка `Загрузить фото`.

2. Для отключенных кнопок показывать понятную причину:
   - нет прав;
   - нет текущего абонемента;
   - клиент в архиве;
   - действие недоступно для текущего статуса клиента.

3. Заменить общий текст про management-роли на подсказки около конкретных действий.

## Responsive

1. Проверить detail/form сценарии на tablet и mobile после введения вкладок.
2. Убедиться, что действия карточки не переносятся хаотично и не перекрывают друг друга.

## Проверки

После реализации запускать:

```bash
cd frontend && npm run lint
cd frontend && npm run build
```

Для заметных UX-изменений добавить или обновить Playwright-тесты на:

- поиск клиента;
- фильтры списка;
- открытие карточки клиента;
- сохранение фильтров при возврате;
- основные быстрые действия;
- мобильный режим карточки клиента.
