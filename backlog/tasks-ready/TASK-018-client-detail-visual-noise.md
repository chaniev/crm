# TASK-018: Уменьшить визуальный шум в client detail/form сценариях

## Status
ready

## Goal
Detail и form сценарии клиента выглядят плотнее и понятнее, без пустых блоков и повторяющихся статусов.

## Context
Нужно проверить специализированные карточки client form/detail на лишние пустые блоки, большой просмотр фото показывать только если фото уже загружено или пользователь явно открыл просмотр, убрать повторяющиеся статусы и сделать данные клиента более плотными.

## User role
администратор / тренер

## Problem
Карточка и формы клиента содержат лишние блоки, повторяющиеся состояния и слишком заметный просмотр фото.

## Scope
- Просмотреть client form/detail компоненты.
- Убрать или свернуть пустые декоративные блоки.
- Показывать большой просмотр фото только при наличии фото или явном открытии.
- Убрать повторяющиеся статусы.
- Сохранить читаемость данных.

## Out of scope
- Изменение модели фото клиента.
- Изменение API клиента.
- Полная переработка вкладок.

## Constraints
- Не скрывать важные warnings и validation states.
- Не дублировать одно и то же состояние в нескольких блоках.
- UI должен оставаться responsive.

## Acceptance criteria
- [ ] Пустые блоки в detail/form убраны или заменены полезным состоянием.
- [ ] Большой просмотр фото не занимает место без причины.
- [ ] Повторяющиеся статусы убраны.
- [ ] Данные стали плотнее без ухудшения readability.

## Test checklist
- [ ] Запустить `cd frontend && npm run lint`.
- [ ] Запустить `cd frontend && npm run build`.
- [ ] Вручную проверить клиента с фото и без фото.
- [ ] Вручную проверить form/detail на mobile.
