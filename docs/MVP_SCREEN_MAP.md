## 2. Рекомендуемая навигация по ролям

`Главный тренер`
- Основная навигация: `Главная`, `Посещения`, `Клиенты`, `Группы`, `Пользователи`, `Журнал действий`
- Профильное меню: `Смена пароля`, `Выход`
- Landing after login: `Главная`

`Администратор`
- Основная навигация: `Главная`, `Клиенты`, `Группы`, `Журнал действий`
- Профильное меню: `Смена пароля`, `Выход`
- Landing after login: `Главная`

`Тренер`
- Основная навигация: `Посещения`, `Клиенты`
- Профильное меню: `Смена пароля`, `Выход`
- Landing after login: `Посещения`

`Общее правило`
- Если `MustChangePassword = true`, пользователь любого типа сначала попадает в `Смену пароля` в forced-режиме без рабочего shell.

## 3. Матрица экранов

| Экран | Роли | Основные действия |
|---|---|---|
| `Авторизация` | Главный тренер, Администратор, Тренер | Ввести логин и пароль, войти, увидеть ошибку входа |
| `Смена пароля` | Главный тренер, Администратор, Тренер | В forced-режиме задать новый пароль и продолжить вход; в utility-режиме сменить пароль из профиля |
| `Главная` | Главный тренер, Администратор | Просмотреть клиентов с окончанием абонемента менее чем через 10 дней, открыть карточку клиента, обновить список |
| `Список клиентов` | Главный тренер, Администратор, Тренер | Искать, фильтровать, открывать карточку клиента; для Главного тренера и Администратора также создавать, редактировать, архивировать и возвращать из архива |
| `Создание клиента` | Главный тренер, Администратор | Заполнить данные клиента, загрузить фото, назначить группы, сохранить |
| `Редактирование клиента` | Главный тренер, Администратор | Менять данные клиента, фото, группы, статус, архивировать или возвращать в активные |
| `Карточка клиента` | Главный тренер, Администратор, Тренер | Просматривать данные клиента; для Главного тренера и Администратора также оформлять абонемент, продлевать, исправлять, отмечать оплату; для Тренера смотреть только доступные группы и историю посещений по своим группам |
| `Список групп` | Главный тренер, Администратор | Просматривать группы, открывать создание и редактирование |
| `Создание группы` | Главный тренер, Администратор | Создать группу, задать расписание и назначить тренеров |
| `Редактирование группы` | Главный тренер, Администратор | Изменить группу, расписание и состав тренеров |
| `Экран посещений` | Главный тренер, Тренер | Выбрать группу и дату, отметить или изменить посещение, увидеть предупреждение по абонементу, работать с пустым состоянием при отсутствии групп |
| `Список пользователей` | Главный тренер | Просматривать пользователей, открывать создание и редактирование |
| `Создание пользователя` | Главный тренер | Создать пользователя с ролью, логином, паролем, активностью и флагом обязательной смены пароля |
| `Редактирование пользователя` | Главный тренер | Редактировать роль, активность и флаг обязательной смены пароля; логин только read-only |
| `Журнал действий` | Главный тренер, Администратор | Фильтровать по периоду, пользователю, типу действия и типу объекта, просматривать детали записи |

## 4. UI-notes: что должно отличаться внутри одного экрана

`Список клиентов`
- Один экран для всех ролей.
- `Главный тренер` и `Администратор` видят полный поиск и фильтры, телефон, статус клиента и CRUD-действия.
- `Тренер` видит только клиентов своих групп, без телефона, без контактных лиц и без CRUD.
- Для `Тренера` в списке остаются только безопасные индикаторы: фото, ФИО, доступные группы, предупреждение о проблеме с абонементом и иконка отсутствия оплаты.
- По уточнению заказчика отметка посещения находится только на `Экране посещений`.

`Карточка клиента`
- Один экран для всех ролей.
- `Главный тренер` и `Администратор` видят полную карточку: контакты, группы, текущий абонемент, оплату, историю посещений.
- `Тренер` видит только фото, ФИО, доступные ему группы и историю посещений по этим группам.
- Полные данные абонемента, оплаты, телефоны и контактные лица для `Тренера` не рендерятся вообще.

`Экран посещений`
- Один экран для `Главного тренера` и `Тренера`.
- Различие только в scope доступа: `Главный тренер` работает с любой группой, `Тренер` только с назначенными группами.
- Экран остается mobile-first и является главным рабочим экраном роли `Тренер`.

`Смена пароля`
- Один экран с двумя режимами, а не два независимых модуля.
- Forced-режим блокирует доступ к рабочему shell до успешной смены пароля.

`Главная`
- Один смысл экрана для обеих ролей: только список клиентов с окончанием абонемента менее чем через 10 дней.
- Не превращать экран в широкий дашборд со вторичными виджетами.

`Формы create/edit`
- `Создание` и `Редактирование` остаются отдельными route-level экранами по ТЗ.
- Внутри реализации допустим общий scaffold формы и переиспользуемые секции.

## 5. Что сознательно не нужно делать отдельными экранами в MVP

- `Абонемент`
- `Оплата`
- `История посещений`
- `История абонемента`
- `Карточка группы`
- Отдельные ролевые копии `Списка клиентов`
- Отдельные ролевые копии `Карточки клиента`
- Отдельные ролевые копии `Экрана посещений`
- Отдельный большой раздел `Смена пароля`
- Отдельный экран `Архив клиентов`
  Архивные клиенты должны открываться через фильтр в `Списке клиентов`.
- Отдельный экран загрузки фото клиента
  Загрузка и замена фото должны быть частью формы клиента или карточки клиента.

## 6. Короткие продуктовые фиксации для реализации

- `Главный тренер` и `Администратор` после входа идут на `Главную`.
- `Тренер` после входа идет на `Экран посещений`.
- У `Тренера` история посещений остается только блоком внутри `Карточки клиента`.
- По уточнению заказчика элементы отметки посещения не выводятся в `Списке клиентов`, даже если ранняя формулировка ТЗ допускает иное чтение.
- У `Тренера` нет ни одного экрана, где можно редактировать клиента, группу, пользователя, абонемент или оплату.
- Backend остается источником истины по доступам; скрытие пунктов меню не заменяет серверную проверку прав.
