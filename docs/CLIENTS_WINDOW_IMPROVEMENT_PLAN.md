# План улучшения окна клиентов

## Цель

Сделать раздел клиентов рабочим CRM-инструментом: пользователь должен быстро находить клиента, понимать его состояние и выполнять ключевые действия без лишнего скролла и переходов.

Сейчас экран выглядит скорее как длинная карточка одного клиента. Для ежедневной работы тренера или администратора важнее быстрый обзор списка, состояние абонемента, оплата, последнее посещение и быстрые действия.

Визуальные HTML-макеты:

- первый вариант: [CLIENTS_WINDOW_TARGET_MOCKUP.html](CLIENTS_WINDOW_TARGET_MOCKUP.html);
- компактная версия с уменьшенной верхней частью: [CLIENTS_WINDOW_TARGET_MOCKUP_V2.html](CLIENTS_WINDOW_TARGET_MOCKUP_V2.html).
- V3 с главным списком и легким предпросмотром клиента: [CLIENTS_WINDOW_TARGET_MOCKUP_V3.html](CLIENTS_WINDOW_TARGET_MOCKUP_V3.html).
- V3 с примером загруженной фотографии клиента: [CLIENTS_WINDOW_TARGET_MOCKUP_V3_PHOTO.html](CLIENTS_WINDOW_TARGET_MOCKUP_V3_PHOTO.html).
- V4 с рабочей таблицей как главным фокусом и облегченным предпросмотром: [CLIENTS_WINDOW_TARGET_MOCKUP_V4.html](CLIENTS_WINDOW_TARGET_MOCKUP_V4.html).
- V5 после UX-разбора: меньше второго уровня в гриде, крупнее фото, меньше badge-шума и действий в строках: [CLIENTS_WINDOW_TARGET_MOCKUP_V5.html](CLIENTS_WINDOW_TARGET_MOCKUP_V5.html).
- V6 после разбора текущего экрана: таблица как основной рабочий слой, более легкий сайдбар, плотные строки, спокойная типографика, единая система сигналов и действий: [CLIENTS_WINDOW_TARGET_MOCKUP_V6.html](CLIENTS_WINDOW_TARGET_MOCKUP_V6.html), handoff: [CLIENTS_WINDOW_TARGET_MOCKUP_V6_SPEC.md](CLIENTS_WINDOW_TARGET_MOCKUP_V6_SPEC.md).
- V7 после уточнения, что долг и заморозка не актуальны: компактный верх без пассивных сводок, единый поиск, частые фильтры и quick-срезы как рабочая панель списка: [CLIENTS_WINDOW_TARGET_MOCKUP_V7.html](CLIENTS_WINDOW_TARGET_MOCKUP_V7.html), handoff: [CLIENTS_WINDOW_TARGET_MOCKUP_V7_SPEC.md](CLIENTS_WINDOW_TARGET_MOCKUP_V7_SPEC.md).

## Этап 1. Переработать список клиентов

1. Сделать полноценный список или таблицу клиентов.
2. Добавить ключевые колонки:
   - ФИО;
   - телефон;
   - статус;
   - текущий абонемент;
   - дата окончания абонемента;
   - остаток занятий;
   - состояние абонемента;
   - группа;
   - последнее посещение.
3. Добавить быстрые действия в строке клиента:
   - открыть карточку;
   - отметить посещение;
   - оформить или продлить абонемент;
   - принять оплату;
   - архивировать.
4. Добавить поиск по ФИО и телефону.
5. Добавить фильтры:
   - активные;
   - архивные;
   - без абонемента;
   - абонемент скоро закончится;
   - по группе.

## Этап 2. Сделать удобную карточку клиента

1. Сжать верхний блок карточки клиента.
2. Вынести наверх самое важное:
   - ФИО;
   - телефон;
   - статус;
   - текущий абонемент;
   - дата окончания абонемента;
   - последнее посещение;
   - группа.
3. Добавить заметные основные действия:
   - отметить посещение;
   - новый абонемент;
   - принять оплату;
   - добавить в группу;
   - редактировать;
   - отправить в архив.
4. Убрать лишний поясняющий текст, который не помогает выполнить действие.
5. Сделать возврат к списку клиентов с сохранением поиска, фильтров и позиции в списке.

## Этап 3. Уменьшить визуальный шум

1. Сделать фото клиента компактным.
2. Большой просмотр фото показывать только если фото уже загружено или пользователь открыл просмотр.
3. Убрать повторяющиеся статусы, например повтор "Активный" в разных блоках.
4. Сократить пустые карточки, которые занимают много места без данных.
5. Показывать данные клиента более плотно: не как огромные поля, а как компактный информационный блок.

## Этап 4. Разделить карточку на вкладки

Предлагаемая структура:

1. Обзор:
   - ключевые данные;
   - текущий абонемент;
   - последние посещения;
   - быстрые действия.
2. Абонемент и оплата:
   - текущий абонемент;
   - история абонементов;
   - оплаты;
   - продление и исправление.
3. Посещения:
   - история посещений;
   - фильтр по периоду;
   - отметка посещения.
4. Группы:
   - текущие группы;
   - добавление в группу;
   - удаление из группы.
5. Контакты:
   - контактные лица;
   - телефон;
   - дополнительные контакты.
6. История:
   - изменения карточки;
   - версии абонемента;
   - служебные события.

## Этап 5. Сделать пустые состояния полезными

Пустые блоки должны не только сообщать об отсутствии данных, но и предлагать следующее действие:

- "Нет абонемента" -> кнопка "Оформить абонемент".
- "Нет посещений" -> кнопка "Отметить первое посещение".
- "Клиент не в группе" -> кнопка "Добавить в группу".
- "Контакты не добавлены" -> кнопка "Добавить контакт".
- "Фото не загружено" -> компактная кнопка "Загрузить фото".

## Этап 6. Разобраться с правами и отключенными кнопками

1. Для отключенных кнопок показывать понятную причину.
2. Разделить причины недоступности:
   - нет прав;
   - нет текущего абонемента;
   - клиент в архиве;
   - действие недоступно для текущего статуса клиента.
3. Заменить общий бейдж "Только для management-ролей" на подсказки около конкретных действий.

## Этап 7. Responsive и мобильный режим

1. На desktop рассмотреть режим "список слева, карточка справа".
2. На tablet оставить список и карточку отдельными экранами, но с быстрым возвратом.
3. На mobile использовать вкладки и компактные блоки без горизонтальных таблиц.
4. Проверить, что кнопки действий не переносятся хаотично и не перекрывают друг друга.

## Этап 8. Проверки

Минимальный набор проверок после реализации:

```bash
cd frontend && npm run lint
cd frontend && npm run build
cd frontend && npm run test:e2e -- e2e/users.spec.ts
```

Дополнительно стоит добавить или обновить Playwright-тесты на:

- поиск клиента;
- фильтры списка;
- открытие карточки клиента;
- сохранение фильтров при возврате из карточки;
- основные быстрые действия.

## Приоритет внедрения

1. Сначала список клиентов: поиск, фильтры, рабочая таблица.
2. Затем компактная шапка карточки и быстрые действия.
3. Затем вкладки и уменьшение пустых блоков.
4. Затем права, подсказки и визуальная полировка.
5. Затем расширенные сценарии оплаты, истории и групп.

## Ожидаемый результат

Пользователь открывает раздел "Клиенты" и сразу видит рабочий список, проблемных клиентов, состояние абонементов и быстрые действия. Карточка клиента становится компактным операционным экраном, а не длинной страницей с большим количеством пустых блоков.
