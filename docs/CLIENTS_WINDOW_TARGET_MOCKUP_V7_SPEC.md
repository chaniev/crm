# UX-спецификация V7: экран "Клиенты"

Основание: V7 уточняет V6 после продуктового решения, что долг и заморозка не являются актуальными сценариями этого окна. Верх экрана должен работать как панель управления списком, а не как мини-дашборд.

Макет: [CLIENTS_WINDOW_TARGET_MOCKUP_V7.html](CLIENTS_WINDOW_TARGET_MOCKUP_V7.html).

## 1. Главный сценарий

Пользователь открывает раздел "Клиенты", быстро находит клиента по имени или телефону, сужает список частыми фильтрами и понимает ближайшее действие прямо в строке.

Переход в полную карточку нужен для редактирования, истории и редких сценариев.

## 2. Верх экрана

Структура сверху вниз:

1. Компактная глобальная шапка:
   - бренд;
   - основная навигация;
   - профиль.
2. Заголовок раздела:
   - `Клиенты`;
   - счетчик результата, например `48 активных из 312`;
   - primary action `Новый клиент`.
3. Toolbar списка:
   - `Поиск` с placeholder `Имя или телефон`;
   - `Статус`: `Активные`, `Все`, `Архив`;
   - `Группа`;
   - `Еще фильтры`;
   - quiet action `Обновить`.
4. Quick-фильтры:
   - `Без абонемента`;
   - `Скоро закончится`;
   - `Без группы`;
   - `Пробные`.

Не показывать пассивные сводки вроде `требуют внимания`, если они не являются кликабельными фильтрами.

## 3. Что убрать из V7

- Отдельный select `Сигнал`.
- Chips с дефолтными значениями вроде `Сигнал: любой`.
- Долг и заморозку во всех местах: summary, фильтры, строки списка, preview.
- Дублирование фильтров в отдельной строке `list-meta`, если активных нестандартных фильтров нет.

## 4. Список

Оставить 5 колонок:

- `Клиент`;
- `Статус и абонемент`;
- `Следующий шаг`;
- `Группа`;
- `Визит`.

Колонка `Следующий шаг` показывает одно главное действие или нейтральное состояние. Примеры:

- `Продлить`;
- `Оформить`;
- `Предложить`;
- `В группу`;
- `Планово`.

Не смешивать несколько равнозначных кнопок в одной строке.

## 5. Preview

Правый preview остается короткой контекстной панелью:

- клиент;
- телефон;
- статус;
- блок `Нужно сейчас`;
- 4-5 ключевых фактов;
- 3 последних события;
- действия `Открыть карточку`, `Визит`, `Редактировать`.

Если срочного действия нет, показывать нейтральное состояние `Планово` или `Ничего срочного`.

## 6. Состояния

Loading:

- toolbar не меняет высоту;
- список показывает skeleton rows;
- счетчик обновляется через `aria-live="polite"`.

Empty после поиска или фильтров:

- заголовок `Клиенты не найдены`;
- текст `Попробуйте изменить поиск или сбросить фильтры.`;
- действие `Сбросить фильтры`.

Empty first-run:

- заголовок `Клиентов пока нет`;
- текст `Создайте первую карточку клиента.`;
- действие `Новый клиент`.

Error:

- inline alert над списком;
- текст `Не удалось загрузить клиентов`;
- действие `Повторить`.

## 7. Responsive

Desktop:

- page header в 1 ряд;
- toolbar в 1 ряд;
- quick-фильтры во 2 ряд;
- preview справа.

Tablet:

- page header остается компактным;
- toolbar переносится в 2 ряда;
- preview скрывается или открывается как отдельная панель.

Mobile:

- `Клиенты` и `Новый клиент` сверху;
- поиск на всю ширину;
- статус отдельной строкой;
- quick-фильтры горизонтальным скроллом;
- список карточками;
- выбранный клиент открывается как отдельный screen/sheet, а не длинным preview под таблицей.
