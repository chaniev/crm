# AGENTS.md

## Назначение

Этот файл задает краткие правила работы с репозиторием и направляет к подробным документам.

## Порядок источников истины

1. `docs/MVP-ТЗ.md` — бизнес-требования, роли, ограничения и критерии приемки.
2. `docs/IMPLEMENTATION_PLAN.md` — этапы, архитектурные решения, текущий статус и порядок реализации.
3. `AGENTS.md` — краткий workflow выполнения задачи в этом репозитории.
4. `docs/AGENT_ROUTING.md` — выбор профильного агента по типу задачи и этапу плана.
5. `docs/DELIVERY_CHECKLIST.md` — чеклист старта, проверки и завершения шага.

## Короткий workflow

- Для каждой задачи сначала читать соответствующий этап в `docs/IMPLEMENTATION_PLAN.md` и связанные ограничения в `docs/MVP-ТЗ.md`.
- Делать минимальное безопасное изменение, которое продвигает один рабочий пользовательский сценарий.
- Перед реализацией фиксировать границу задачи: entry point, затронутые данные, права доступа, внешние зависимости.
- Если меняются контракты, структура проекта или важные технические решения, обновлять документы в репозитории.
- Если каркас проекта отсутствует или сломан базовый запуск, приоритетом становится `Этап 0. Подготовка проекта`.

## Использование агентов

- По умолчанию для задачи автоматически подбирать наиболее подходящего профильного агента.
- Если ни один агент не подходит по зоне ответственности, задача выполняется локально.
- Многослойные задачи делить по границам: backend, frontend, тесты, контейнеризация, UI/UX.
- Делегировать агенту ограниченную по области задачу с понятной зоной ответственности.
- Если нужен только implement-ready UI-вариант без кодинга, сначала использовать `ui-designer`.

## Доступные агенты

- По умолчанию используется автоматически подобранный профильный агент; локальное выполнение — fallback, если подходящего агента нет.
- `csharp-developer` — доменная модель, прикладные сервисы, DTO, транзакции, версионирование сущностей и бизнес-логика `.NET`.
- `dotnet-core-expert` — `ASP.NET Core` API, middleware, авторизация, CSRF, DI, конфигурация и HTTP boundary.
- `react-specialist` — компоненты `React`, формы, состояние, маршруты, доступность и поведение экранов.
- `ui-designer` — implement-ready UI/UX решения, состояния `loading/empty/error`, фокус и keyboard flow.
- `test-automator` — автотесты, regression coverage, фикстуры, test harness и стабилизация тестовой инфраструктуры.
- `docker-expert` — `Dockerfile`, `docker-compose`, container runtime, health checks и эксплуатационное поведение контейнеров.

## Непереговоримые инварианты проекта

- Проверка прав и ролевых ограничений остается на backend; frontend не является источником истины по доступам.
- Для тренера доступ к клиентам и данным ограничен назначенными группами и разрешенным набором полей.
- `ClientMembership` требует аккуратного версионирования и соблюдения правила "не более одной текущей записи".
- Проблемы с абонементом не блокируют отметку посещения, а только создают предупреждение.
- `AuditLog` остается append-only и не хранит пароль или хеш пароля.
