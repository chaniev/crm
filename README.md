# CRM для спортивного зала

Репозиторий MVP CRM-системы для учета клиентов спортивного зала, групп, тренеров, абонементов, оплат и посещений.

На текущем этапе репозиторий содержит проектную документацию и план реализации. Исходный код приложения еще не добавлен.

## Статус проекта

Статус: проектирование MVP.

Цель MVP:

- вести базу клиентов;
- объединять клиентов в группы;
- назначать тренеров на группы;
- учитывать абонементы и оплаты;
- отмечать посещения тренировок;
- хранить историю изменений абонементов и историю посещений;
- вести журнал действий пользователей;
- разграничивать доступ по ролям.

## Что входит в MVP

В MVP предусмотрены следующие блоки:

- авторизация по логину и паролю;
- обязательная смена пароля при первом входе;
- роли: Главный тренер, Администратор, Тренер;
- управление пользователями;
- управление клиентами, статусами и контактными лицами;
- управление группами и назначением тренеров;
- учет абонементов клиента через `ClientMembership`;
- отметка оплат;
- экран отметки посещений;
- история посещений клиента;
- журнал действий пользователей;
- поиск, фильтрация и адаптивный интерфейс.

## Роли и доступ

### Главный тренер

- полный доступ к пользователям, клиентам, группам, абонементам и посещениям;
- может отмечать посещения в любых группах;
- имеет доступ к журналу действий.

### Администратор

- управляет клиентами, группами, абонементами и оплатами;
- не управляет пользователями;
- не может отмечать посещения;
- имеет доступ к журналу действий.

### Тренер

- работает только со своими назначенными группами;
- может отмечать посещения и редактировать их задним числом только в своих группах;
- видит ограниченный набор данных по клиенту;
- не имеет доступа к управлению пользователями, группами, абонементами и журналу действий.

## Ключевые бизнес-правила

- клиент может быть создан без текущего абонемента;
- у клиента может быть не более одной текущей записи `ClientMembership`;
- абонемент редактируется только из карточки клиента, а не как отдельный пользовательский раздел;
- разовый абонемент автоматически списывается после одного отмеченного посещения;
- проблемы с абонементом не блокируют отметку посещения, а только показывают предупреждение;
- тренер не должен получать через API телефон, контактных лиц, полные данные абонемента и детали оплаты;
- журнал действий хранит историю изменений в append-only формате.

## Планируемый стек

- backend: `C# / .NET 10`;
- frontend: `React`;
- база данных: `PostgreSQL`;
- контейнеризация: `Docker`;
- локальный запуск: `docker-compose`.

## Безопасность MVP

- cookie-based авторизация для браузерного frontend;
- `JWT` для браузерного frontend не используется;
- авторизационная cookie должна быть `HttpOnly`;
- в production cookie должна быть `Secure`;
- изменяющие запросы должны быть защищены через CSRF;
- проверка прав доступа выполняется на backend.

## Первый пользователь

При развертывании системы должен автоматически создаваться первый пользователь:

- роль: Главный тренер;
- пароль: `12345678`;
- обязательная смена пароля при первом входе: включена;
- логин задается через конфигурацию или используется значение по умолчанию.
