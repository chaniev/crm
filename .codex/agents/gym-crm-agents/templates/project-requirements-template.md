# Project context
Опиши продукт, текущую стадию, кто заказчик, что уже есть.

# Business goals
Например:
- снизить время операций администратора
- сократить обучение новых сотрудников
- уменьшить количество ошибок при записи и продлении

# Primary users
Опиши роли:
- администраторы
- тренеры
- вторичные роли, если есть

# Functional requirements
Список функций. Например:
- поиск клиента
- check-in
- продажа и продление абонементов
- расписание тренировок
- отметка посещения
- заметки тренера
- платежи

# Non-functional requirements
Например:
- быстрый отклик поиска
- работа на десктопе и планшете
- доступность
- локализация

# Constraints
Например:
- сроки
- существующий backend
- обязательные модули
- ограничения на стек

# Existing screens
Если есть, вставь ссылки, скриншоты, описание проблем.

# Brand preferences
Если есть фирменные цвета, стиль, пожелания.

# Technical stack
Например:
- React + TypeScript
- existing API
- auth provider

# Design references
Примеры интерфейсов, которые нравятся / не нравятся.

# Accessibility requirements
Контраст, размер шрифта, работа с клавиатурой и т.д.

# Performance targets
Например:
- поиск клиента < 1 сек
- check-in за 1 действие
